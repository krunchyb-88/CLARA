# Contributing to CLARA

Thank you for your interest in contributing to CLARA. This document sets out the guidelines, scope, and contributor terms for this project.

Please read this document in full before submitting any contribution.

---

## What Kind of Contributions Are Welcome

CLARA is a structured system prompt and accompanying documentation. Contributions that improve clarity, correctness, safety, or utility are welcome. Specifically:

| Type | Examples |
|---|---|
| **Corrections** | Fixing factual errors in evidence citations, outdated guideline references, broken links |
| **Clarity improvements** | Rephrasing instructions that are ambiguous or inconsistent |
| **Safety enhancements** | Strengthening behavioural constraints, improving Clinical Bridge outputs, tightening scope boundaries |
| **Localisation** | Adapting reference authorities for non-Singapore jurisdictions (e.g. substituting MOH Singapore with a local equivalent), clearly marked as a regional variant |
| **Documentation** | Improving README, DISCLAIMER, or other supporting files |
| **Knowledge File contributions** | Additions or corrections to the CLARA Knowledge File, where published |

---

## What Is Out of Scope

The following contributions will not be accepted:

- Changes that weaken or remove any behavioural constraint, safety rule, or Clinical Bridge requirement
- Changes that expand CLARA's scope to include diagnostic outputs or individual treatment recommendations
- Additions that introduce commercial affiliate links, product endorsements, or promotional content
- Changes that remove or dilute the disclaimer or liability language
- Modifications that remove attribution to the original author
- Contributions that introduce content generated primarily by an AI model without human review and verification by the contributor

---

## How to Contribute

1. **Open an Issue first** for any non-trivial change — describe what you want to change and why. This allows discussion before you invest time in a pull request.
2. **Fork the repository** and make your changes in a feature branch.
3. **Submit a Pull Request** with a clear description of:
   - What was changed
   - Why the change improves CLARA
   - Any sources or references that support the change (especially for evidence-related edits)
4. All pull requests will be reviewed by the maintainer. Review may take time — this is a solo-maintained project.

## Code of Conduct

This project is maintained in the spirit of good-faith collaboration. Contributors are expected to:

- Be respectful and constructive in all communications
- Engage with evidence and reasoning, not authority or assertion
- Acknowledge uncertainty rather than overstate confidence
- Prioritise user safety above all other considerations

Contributions or communications that are dismissive, hostile, or made in bad faith will not be engaged with and may result in removal from the project.

---

## A Note on Health Content

CLARA operates in health and medical adjacent territory. This creates a heightened responsibility for contributors. If you are proposing a change to any evidence citation, clinical claim, drug interaction note, or behavioural constraint:

- Cite your source explicitly in the pull request
- Prefer primary sources (systematic reviews, guideline documents, peer-reviewed trials) over secondary or tertiary sources
- If you are a healthcare professional and your contribution draws on clinical expertise, you are welcome to note this — but the same source standards apply

No contribution removes the user's obligation to verify information with a qualified healthcare professional.
